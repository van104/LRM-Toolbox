```markdown
1. **调研与准备**：首先浏览 `docs/` 目录下的 Markdown 文档，了解项目的开发规范、架构设计及现有工具的实现逻辑。
2. **定义元数据**：在 `src/data/tools/` 目录下找到对应的分类 JSON 文件，按格式添加新工具的配置信息（如 ID、名称、描述、图标等），这是工具在列表页展示的依据。
3. **逻辑复用调研**：浏览 `src/composables` 目录下的全局函数，优先复用已有的逻辑实现（例如：使用 `useCopy` 实现一键复制，使用 `useDownload` 处理文件导出等），确保功能交互的一致性。
4. **组件开发**：在 `src/components/tools/` 目录下创建工具的主组件。
   - **样式规范**：必须遵循项目统一的 UI 规范（如使用 Naive UI 或 Tailwind CSS），确保视觉风格与工具箱整体保持一致。
   - **组件拆分**：对于功能复杂的工具，务必将其拆分为多个子组件或将核心逻辑抽离至独立的 `composable` 中，避免单文件代码量过大。
5. **调试与验证**：在本地环境预览工具，检查响应式布局适配以及在不同输入情况下的健壮性。
```
